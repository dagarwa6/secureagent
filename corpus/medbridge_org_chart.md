## Security Governance Structure (Current State)

**Informal / Ad-Hoc — No Formal Security Governance:**

- No Security Steering Committee
- No formal risk management process
- Security decisions made reactively by IT Director
- No documented security strategy or multi-year roadmap
- Security budget is a subset of IT budget with no dedicated line item
- No board-level cybersecurity reporting; security incidents not reported to board unless regulatory (HIPAA breach)
- No security KPIs tracked or reported to executive team

**Accountability gaps:**
- Vendor security assessments: No owner
- Security awareness training: HR "owns" it but defers to IT; no formal program
- Incident response: IT Director handles; no documented escalation path to C-suite
- Risk register: Does not exist; no formal tracking of IT risks

## Key Organizational Risk Factors

1. **No CISO:** Security is not represented at the executive level; budget and priority decisions lack security perspective
2. **Dual-role HIPAA Officer:** Compliance team stretched thin; risk analysis overdue
3. **Vacant Security Analyst:** Only planned security resource has been unfilled 7 months
4. **Siloed Clinical IT:** Distributed clinical IT staff make independent decisions outside central governance
5. **Low security training completion:** 39% of staff have not completed training; 34% phishing click rate indicates high social engineering susceptibility
6. **No security steering committee:** No cross-functional forum for security decision-making or escalation
